# Hermes Automation Design

## Purpose

Workhub의 헤르메스를 단순 채팅창이 아니라 업무 AI 허브로 확장한다. 사용자는 AI 업무채팅에서 외부 데이터 검색, Workhub 데이터 조회, 메일/공지 초안 작성, 승인 후 실행 업무를 요청할 수 있다.

핵심 원칙은 안전한 자동화다. 헤르메스는 위험도가 낮은 조회와 요약은 바로 수행할 수 있지만, 메일 발송, DB 변경, 대량 처리, 파일 삭제/덮어쓰기 같은 업무는 반드시 미리보기와 관리자 승인을 거친 뒤 실행한다.

## Scope

1차 범위는 다음 기능으로 제한한다.

- AI 업무채팅에서 외부 검색/업무 요청을 접수한다.
- 요청을 작업 유형으로 분류한다.
- 실행 전 미리보기 화면을 만든다.
- 승인 대기 작업 큐를 저장한다.
- 관리자 승인 후 허용된 작업만 실행한다.
- 모든 검색, 초안, 승인, 실행 결과를 헤르메스 작업내역에 저장한다.

1차에서 실제 실행 가능한 작업은 다음으로 제한한다.

- 외부 검색 결과 요약
- Workhub 내부 데이터 조회 요약
- 메일 초안 생성
- 공지사항 초안 생성
- CS/매출/통합관리대장 현황 요약
- 백업 상태 확인

2차 이후에 실제 메일 발송, DB 일괄 변경, 백업 생성, 공지 자동 등록을 승인형 실행 작업으로 추가한다.

## User Flow

1. 사용자가 헤르메스 AI 업무채팅에 업무를 입력한다.
2. Workhub가 요청을 헤르메스에 전달한다.
3. 헤르메스가 필요한 도구를 판단한다.
4. 조회/검색이면 결과와 출처를 표시한다.
5. 실행이 필요한 작업이면 미리보기를 만든다.
6. 사용자는 `승인 후 실행`, `초안만 저장`, `수정 요청`, `취소` 중 하나를 선택한다.
7. 승인된 작업만 실행된다.
8. 작업 결과와 로그가 작업내역에 저장된다.

## Permissions

- 일반 사용자
  - AI 질문
  - 외부 검색 요청
  - 내부 데이터 조회 요약
  - 초안 생성

- 관리자
  - 메일 발송 승인
  - 공지 등록 승인
  - CS 요청 실행 승인
  - 백업 생성 승인

- 최고 관리자
  - DB 대량 수정 승인
  - 삭제/덮어쓰기 승인
  - 외부 연동 설정 변경
  - 헤르메스 API/브리지 설정 변경

## Data Model

새 저장 데이터는 기존 `config/hermes_history.jsonl`과 별도 승인 큐 파일을 함께 사용한다.

- `config/hermes_history.jsonl`
  - 사용자 질문
  - 헤르메스 답변
  - 검색 요약
  - 승인/실행 결과

- `config/hermes_pending_actions.json`
  - 승인 대기 작업 목록
  - 작업 유형
  - 요청자
  - 미리보기 내용
  - 위험도
  - 승인 상태
  - 실행 결과

DB가 아닌 JSON 파일로 시작하는 이유는 현재 헤르메스 작업내역 구조와 맞고, 1차 구현 범위를 작게 유지하기 위해서다. 실행 이력이 늘어나거나 검색 로그가 많아지면 이후 SQLite 테이블로 이전한다.

## Tool Types

1차 도구 유형은 다음으로 고정한다.

- `external_search`: 외부 웹 검색 및 출처 요약
- `workhub_query`: Workhub 내부 데이터 조회
- `draft_email`: 메일 초안 작성
- `draft_notice`: 공지사항 초안 작성
- `status_check`: 백업/VPS/연동 상태 확인
- `pending_action`: 승인 대기 작업 생성

각 도구는 결과를 사용자에게 보여주되, 실제 변경 작업은 `pending_action`을 거친다.

## Safety Rules

- 외부 검색 결과에는 출처 링크 또는 확인 경로를 함께 표시한다.
- 검색 결과만으로 DB를 수정하지 않는다.
- 메일 발송, DB 수정, 파일 삭제, 대량 변경은 자동 실행하지 않는다.
- 승인 전 미리보기에는 대상, 내용, 영향 범위, 위험도를 표시한다.
- 승인자, 실행 시각, 실행 결과를 작업내역에 저장한다.
- 실패한 작업은 재시도 버튼보다 실패 원인과 수동 확인 경로를 먼저 보여준다.

## UI Design

헤르메스 화면은 현재 4개 탭을 유지한다.

- AI 업무채팅
- 자동화 요청
- 작업내역
- 관리자 설정

AI 업무채팅 탭은 좌우 레이아웃을 유지한다.

- 왼쪽: 사용자 입력, 빠른 요청 버튼, 요청 유형 표시
- 오른쪽: 답변, 검색 출처, 실행 미리보기, 승인 버튼

자동화 요청 탭에는 승인 대기 목록을 추가한다.

- 작업명
- 요청자
- 위험도
- 상태
- 미리보기
- 승인/취소 버튼

작업내역 탭에는 필터를 추가한다.

- 전체
- 검색
- 초안
- 승인대기
- 실행완료
- 실패

## Backend Design

기존 헤르메스 API를 확장한다.

- `POST /api/hermes-chat`
  - 일반 채팅과 검색/초안 요청을 처리한다.

- `POST /api/hermes-automation`
  - 자동화 요청을 접수하고 승인 대기 작업을 만든다.

- `GET /api/hermes-pending-actions`
  - 승인 대기 작업 목록을 조회한다.

- `POST /api/hermes-pending-actions/approve`
  - 관리자 승인 후 작업을 실행한다.

- `POST /api/hermes-pending-actions/cancel`
  - 승인 대기 작업을 취소한다.

외부 검색은 처음에는 서버 내부 안전 래퍼 함수로 만든다. 실제 검색 공급자는 환경에 따라 바꿀 수 있게 추상화한다.

## Error Handling

- 헤르메스 연결 실패: 연결 설정, 브리지 상태, VPS 컨테이너 상태 확인 문구를 보여준다.
- 외부 검색 실패: 검색 실패 사유와 수동 확인 링크를 보여준다.
- 권한 부족: 필요한 권한 이름을 표시한다.
- 승인 작업 실패: 작업내역에 실패 상태로 저장하고 재실행 전 확인 메시지를 보여준다.

## Testing

최소 검증 항목은 다음이다.

- 헤르메스 메뉴와 탭이 렌더링된다.
- 일반 사용자에게 관리자 실행 버튼이 숨겨진다.
- 자동화 요청이 승인 대기 작업으로 저장된다.
- 승인 대기 작업 목록이 렌더링된다.
- 승인 없는 실행 요청은 거부된다.
- 작업내역에 검색/초안/실행 결과가 저장된다.
- Python 문법 검사를 통과한다.

## Open Decisions

- 외부 검색 공급자는 1차에서 직접 웹 검색 대신 안전한 검색 래퍼로 시작한다.
- 실제 메일 발송과 DB 변경은 2차에서 승인형 실행으로 추가한다.
- 운영 VPS 배포는 로컬 테스트와 사용자 승인 후 진행한다.
